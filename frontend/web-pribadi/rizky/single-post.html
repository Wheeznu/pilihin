Memahami Algoritma Hashing
Tanpa Pusing

📅 14 Mei 2026 ✍️ Rizky Ahmad Nasrulloh
⏱ 8 menit baca

Saat mengerjakan tugas struktur data, seringkali kita terjebak pada pemikiran bahwa semuanya harus diselesaikan dengan pendekatan Object-Oriented Programming (OOP). Padahal, untuk memahami dasar kerja sebuah algoritma, terkadang kembali ke dasar itu jauh lebih baik.

Dalam pengerjaan tugas implementasi hashing kali ini, ada aturan main yang cukup menantang: dilarang menggunakan class. Kita harus menyelesaikan program murni menggunakan pendekatan fungsi prosedural — tidak lebih, tidak kurang.

Kenapa Fungsi Prosedural?

Membangun fungsi hash tanpa class memaksa kita untuk benar-benar memahami bagaimana data mengalir dari satu fungsi ke fungsi lain. Kita tidak bisa menyembunyikan state di dalam objek — semuanya harus eksplisit.

"Kadang batasan (constraints) dalam coding justru membuat kita lebih kreatif dan paham fundamental."

Contoh Snippet Logika

Berikut adalah bayangan kasar bagaimana kita bisa mengelola hash table menggunakan array dan fungsi sederhana di Python, tanpa melibatkan OOP sama sekali: 

# Inisialisasi hash table kosong
hash_table = [None] * 10

# Fungsi prosedural untuk menghitung index
def hitung_hash(nilai):
    return nilai % 10

# Fungsi prosedural untuk memasukkan data
def insert_data(tabel, nilai):
    index = hitung_hash(nilai)
    tabel[index] = nilai

# Fungsi prosedural untuk mencari data
def cari_data(tabel, nilai):
    index = hitung_hash(nilai)
    return tabel[index] == nilai

Collision: Masalah Utama Hashing

Ketika dua nilai berbeda menghasilkan index yang sama, itulah yang disebut collision. Pendekatan paling umum untuk menyelesaikan ini adalah chaining — setiap slot menyimpan list, bukan satu nilai tunggal. 

# Hash table dengan chaining
def create_hash_table(size):
    return [[] for _ in range(size)]

def chaining_insert(tabel, nilai):
    index = nilai % len(tabel)
    tabel[index].append(nilai)

def chaining_search(tabel, nilai):
    index = nilai % len(tabel)
    return nilai in tabel[index]

Pendekatan ini sangat straightforward. Output dan inputnya jelas. Bagi yang sedang belajar algoritma, cobalah sesekali membuang class dan lihat bagaimana kodemu terbentuk murni dari deretan fungsi prosedural — kamu akan lebih paham alur datanya.
